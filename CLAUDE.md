## Folder structure

- `Context/`: the agency knowledge base
- `Clients/`: one folder per client (strategy docs, briefs, transcripts; live work stays in Shade + ClickUp)
- `Marketing/`, `Sales/`, `Operations/`: the agency's own work
- `Production/`: shoot-to-delivery SOPs and naming conventions
- `Skills/`: reusable AI procedures (add as built)
- `_Workshop Materials/`: reference only, the workshop this OS came from

## Output rules

- Australian English. **No em dashes or en dashes, ever.** Use commas, colons, or parentheses.
- Short, direct, scannable. Lead with the answer. No corporate jargon, no AI-slop phrasing.
- Save deliverables as files in the right folder, not just in chat.
- File naming: `[YYMMDD] - [Client] - [Project] - [Doc] - [V].ext` (e.g. `260714 - CNH - Dalby Shoot - Brief - V1.md`)
- Default to Markdown unless another format clearly fits better.

## Safety rules

- Do NOT permanently delete files. Move to an `archives/` folder instead.
- Do NOT send emails, messages, or anything external. Draft only; Steven approves and sends.
- Do NOT paste client-confidential data, credentials, or financial records into this workspace.
- Do NOT change this CLAUDE.md or the context files without proposing the change first.
- If a task is ambiguous or high-risk, stop and ask rather than guess.
